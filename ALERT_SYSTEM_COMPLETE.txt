HOW ALERTS WORK NOW
===================

Flow when user says "HELP":
  1. Create unique alert token + help_command_id
  2. Check Guard 0: Is this same help command? → REJECT if yes
  3. Check Guard 1: Is global cooldown active? → REJECT if yes
  4. Check Guard 2: Is token already processed? → REJECT if yes
  5. Load guardians from config.json
  6. Create alert message with location link
  7. Send to each guardian:
     - Get phone number
     - Send WhatsApp text
     - Log success/failure
  8. Print comprehensive summary:
     - Count successful sends
     - List failed attempts
     - Show overall status
  9. Return response to app

TROUBLESHOOTING MATRIX
=====================

Issue: "NO GUARDIANS FOUND!"
├─ Cause: Guardians not in config.json
├─ Fix: Add guardians to config.json
└─ Terminal: Shows debugging info

Issue: "ConnectionError"
├─ Cause: WhatsApp server not running
├─ Fix: npm start in wa_server folder
└─ Terminal: Shows connection error

Issue: "FAILED - HTTP 401/403"
├─ Cause: WhatsApp authentication failed
├─ Fix: Check WhatsApp server logs
└─ Terminal: Shows HTTP status code

Issue: Alert doesn't send twice
├─ Cause: By design - prevents duplicates
├─ Fix: Wait 20+ seconds
└─ Terminal: Shows cooldown message
