PRODUCTION DEPLOYMENT
=====================

DEPLOYMENT COMMANDS
-------------------

Using Gunicorn:
$ gunicorn --worker-class eventlet -w 1 \
  --bind 0.0.0.0:5000 \
  --log-level info \
  --access-logfile logs/access.log \
  --error-logfile logs/errors.log \
  app:app

Using Docker:
$ docker build -t shakti-alert:1.0.0 .
$ docker run -d -p 5000:5000 \
  -e FLASK_ENV=production \
  -e DATABASE_URL=postgresql://... \
  -v logs:/app/logs \
  shakti-alert:1.0.0

Using Kubernetes:
$ kubectl apply -f deployment.yaml

ROLLBACK PROCEDURE
==================

If deployment fails:

1. Stop current deployment:
   - Kill the process or container
   - Keep previous version running

2. Check error logs:
   - Review logs/errors.log for errors
   - Check application logs

3. Investigate issues:
   - Check database migrations status
   - Verify configuration variables
   - Test health endpoints

4. Rollback:
   - Revert environment variables
   - Revert to previous deployment
   - Re-run database migrations if needed

5. Restart with previous version:
   - Restart application
   - Verify health endpoints
   - Monitor error logs
