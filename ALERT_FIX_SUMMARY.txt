ALERT SENDING LOGIC
===================

When user says "HELP":

1. Create alert with unique token
   ↓
2. Check 3 guards:
   [✓] Is this a duplicate of the same help command?
   [✓] Is global cooldown active? (20 sec min between alerts)
   [✓] Is this token already processed?
   ↓
3. If all pass:
   - Load guardians from config.json
   - Get current location
   - Create alert message with location link
   ↓
4. Send to each guardian:
   - Try WhatsApp
   - Show success/failure for each one
   - Count total sent vs failed
   ↓
5. Print summary:
   - How many sent
   - How many failed
   - Overall status


PREVENTING DUPLICATE ALERTS
===========================

Old system:
- Alert could be sent multiple times if user says "HELP" twice
- Relied only on global cooldown (20 seconds)

New system:
- Each "help" command has unique ID
- If same command tries again: REJECTED
- Protects against accidental double-sends
- Also has global cooldown (20 seconds)
- Also has token-based deduplication

COMMON ISSUES & FIXES
====================

Issue: "NO GUARDIANS FOUND!"
Fix: Add guardians to config.json with correct format

Issue: "ConnectionError"
Fix: Start WhatsApp server (npm start in wa_server)

Issue: Alert doesn't send second time
Fix: Wait 20+ seconds - system prevents rapid alerts

Issue: Phone shows unread but no message
Fix: Check phone's WhatsApp is authenticated

Issue: Status code 400/401/403
Fix: Check WhatsApp server logs for auth issues
