ISSUE 1: Alerts Not Sending at All
==================================

Check these in order:

1️⃣ Is WhatsApp server running?
   Terminal command: netstat -an | findstr 3001
   Expected: Should show listening on port 3001
   Fix if missing: npm start (in wa_server folder)

2️⃣ Are guardians configured?
   File: shakti_alert/config.json
   Should have:
      {
        "guardians": [
          {"name": "Mom", "phone": "+919876543210"},
          {"name": "Dad", "phone": "+919987654321"}
        ]
      }
   Fix: Add guardians with correct format

3️⃣ Check terminal output
   When sending alert, should see:
      ✅ Global cooldown removed - allows multiple help commands
      ✅ All guards passed - Processing alert NOW
      📱 LOADING GUARDIANS FROM CONFIG...
      ✅ Loaded from config: 2 guardian(s)
   
   If missing: Something wrong with alert route

4️⃣ Run debug script
   Command: python DEBUG_ALERT_SYSTEM.py
   Should show: All checks PASS
   Fix: Follow recommendations from script


ISSUE 2: Multiple Alerts Not Working (Still Blocked)
=====================================================

If you say HELP multiple times but only first sends:

Check:

1️⃣ Terminal shows cooldown message?
   Message to see: "✅ Global cooldown removed - allows multiple help commands"
   If missing: Code not updated properly
   Fix: Verify app.py was modified correctly

2️⃣ Are these same or different help commands?
   Same command (duplicate):
      └─ Terminal shows: "ALERT ALREADY SENT FOR THIS HELP COMMAND"
      └─ This is EXPECTED - duplicate prevention
   
   Different commands:
      └─ Terminal shows: "All guards passed - Processing alert NOW"
      └─ Should both send

3️⃣ Check alert token
   Terminal shows: 📋 Token: [unique_token]
   Same token = same command (duplicate)
   Different token = different command (should send)

4️⃣ Still not working?
   Try: Restart Flask backend
   Try: Clear PROCESSED_ALERTS data (temporary storage)
   Run: python DEBUG_ALERT_SYSTEM.py


ISSUE 3: Email Not Sending
===========================

Check each step:

1️⃣ Did you configure email credentials?
   Check .env file exists
   Check has: GMAIL_USER=...
   Check has: GMAIL_APP_PASSWORD=...
   If missing: Add credentials to .env

2️⃣ Is Flask restarted after adding credentials?
   Changes to .env require restart
   Stop Flask app
   Start Flask app again
   Test alert

3️⃣ Terminal shows what about email?
   Expected: [EMAIL] ✅ Sent to your-email@gmail.com
   If shows: [EMAIL] ❌ No credentials found in .env
      → Fix: Add GMAIL_USER to .env
   If shows: [EMAIL] ❌ No recipient email address provided
      → Fix: Recipient email is empty

4️⃣ Check Gmail app password
   Go: https://myaccount.google.com/security
   Generate new app password
   Update .env with new password
   Restart Flask
   Test

5️⃣ Still not working?
   Check Gmail security settings
   Ensure 2FA enabled
   Try: Turn off Gmail less secure settings
   Try: Create new app password and update .env


ISSUE 4: Duplicate Alerts Still Happening
===========================================

This is rare but here's how to check:

1️⃣ Are you sending truly different help commands?
   Alert 1: Time 0s, Token abc123
   Alert 2: Time 5s, Token def456 (DIFFERENT TOKEN = OK to send both)
   Alert 3: Time 8s, Token abc123 (SAME TOKEN = should be rejected)

2️⃣ Check terminal shows:
   For duplicate: "⚠️  ALERT ALREADY SENT FOR THIS HELP COMMAND!"
   For new command: "✅ All guards passed - Processing alert NOW"

3️⃣ Guardians receiving duplicates from single command?
   This shouldn't happen
   Check: Guardian list not duplicated in config.json
   Check: send_whatsapp_text not called twice per guardian
   Fix: Review config.json guardians section

4️⃣ Multiple DIFFERENT commands causing spam?
   This is: Normal behavior (feature, not bug)
   To limit: Add frontend rate limiting
   To limit: Add custom cooldown logic (contact support)


ISSUE 5: Terminal Shows Error
==============================

Common errors and fixes:

Error: "Connection refused on localhost:3001"
   Cause: WhatsApp server not running
   Fix: npm start (in wa_server folder)

Error: "No guardians found!"
   Cause: config.json missing guardians section
   Fix: Add guardians to config.json with correct format

Error: "HTTP 401/403"
   Cause: WhatsApp server authentication failed
   Fix: Check WhatsApp server credentials
   Fix: Restart WhatsApp server

Error: "[EMAIL] No credentials found"
   Cause: GMAIL_USER not in .env
   Fix: Add GMAIL_USER to .env
   Fix: Restart Flask app

Error: "JSON decode error"
   Cause: WhatsApp server not responding properly
   Fix: Restart WhatsApp server
   Fix: Check server logs
